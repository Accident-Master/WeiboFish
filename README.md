# 🐟 WeiboFish: 政务新媒体多智能体与大模型全景仿真沙盘

**WeiboFish** 是一款专为**公共管理与社会科学实证研究**设计的政务新媒体舆情仿真沙盘。
本项目在开发过程中受到了 MiroFish 项目关于多智能体仿真逻辑的启发。WeiboFish 在此基础上，针对政务新媒体场景进行了实证数据校准、引入了 FAISS 稠密向量检索机制（RAG），并对 Agent 推演与智库研判功能进行了深度迭代。特此向原作者与研究团队表示感谢！

---

## ✨ 核心亮点 (Features)

1. **🔬 定量理论基座 (PyTorch)**：内置基于真实数据的多元回归模型参数，通过双向 LSTM 和 RoBERTa 实时提取通报文本的可读性与情绪强度，测算先验的“理论互动概率”。
2. **📚 历史案卷对标 (RAG + FAISS)**：首创“基于历史经验的相似度推理（CBR）”。系统能在毫秒级从 **32万条** 历史政务微博中检索出最相似的真实案卷，并提取当时的真实互动量（转/赞/评）作为大模型的决策依据。
3. **🤖 微观群体行为漏斗 (MAS)**：1000 个具备“年龄、职业、政治倾向、初始信任度”的多样化网民智能体进行推演。首创**【前台显性互动区】与【后台潜意识监控区 (OS)】双轨透视**，直观展现网民“敢怒不敢言”或“盲目跟风”的群体心理学现象。
4. **🧠 智库级内参专报 (DeepSeek-Reasoner)**：在推演结束后，自动调用具备 CoT (思维链) 能力的顶级大模型，结合定量预测、历史案卷与抽样推演结果，生成格式严谨的政务决策内参。

## 🚀 快速开始 (Quick Start)

### 1. 环境依赖与安装

将本项目克隆到本地后，建议使用 Conda 创建虚拟环境：

```bash
cd WeiboFish
pip install -r requirements.txt

```

*(💡 提示：为了加速 FAISS 向量计算，强烈建议安装支持 CUDA 的 PyTorch 版本。)*

### 2. 配置大模型 API 密钥

本项目目前仅支持 DeepSeek 。请前往 DeepSeek 开放平台申请 API Key。

您可以：
1. 打开根目录下的 `app.py`。
2. 找到代码顶部的 `MY_API_KEY = ""`，将您的密钥填入其中。

或者直接在窗口中输入您的API

### 3. 数据准备与向量记忆库构建

为了让智能体拥有历史记忆，您必须先在本地构建 FAISS 向量库。

**获取数据**：您可以将自己的数据命名为 `文章列表汇总.xlsx` 或 `.csv` 放入 `data/` 目录。或者，您也可以使用我们准备的 32 万条开源政务微博数据集：

> **下载链接**: https://pan.baidu.com/s/1HTMfa85D14u-X-94lmfzEQ?pwd=6r92
> **提取码**: 6r92 (下载后放入 `data/` 文件夹)

**构建索引**：在终端运行以下命令。系统将自动下载 BGE 中文小模型并把海量文本压缩为轻量级记忆库。

```bash
python build_vector_db.py

```

*运行成功后，`data/` 目录下将生成 `weibo_memory.index` 和 `weibo_memory_meta.pkl` 两个文件。*

### 4. 启动沙盘

在终端运行以下命令，即可在浏览器中开启可视化沙盘推演：

```bash
streamlit run app.py

```

## 🖥️ 沙盘推演流程指南

启动系统后，在左侧边栏输入您要测试的**“政务新媒体文章”**的元数据，之后在输入框中输入全文，最后点击【🚀 启动全景仿真推演】，系统将依次执行以下四大模块：

1. **第一阶段：传播潜力定量测算**
* 提取文本特征打分（可读性、媒介丰富度等），给出基础传播评价。


2. **第二阶段：历史相似案卷检索 (RAG)**
* 自动匹配库中高度相似的历史真实事件，展示当时真实的网民反馈数据。


3. **第三阶段：多智能体推演实况 (MAS)**
* 观察不同人设的智能体如何阅读通报、参考历史、互相影响，并产生点赞、转发和尖锐内心评论。


4. **第四阶段：政务智库研判专报**
* 系统生成一份严肃的诊断报告，指出文本硬伤，并给出具体的危机公关/文本修改建议。
